# Leave Tracker App

## Overview
A simple leave tracking system for teams to manage leave requests, approvals, and team availability using a shared calendar.

---

## Core Features
- Apply for leave
- Manager approval/rejection
- Shared calendar view
- Leave balance tracking
- Conflict warning for overlapping leaves

---

## UI Expectations
- The primary interface should be a **calendar-based UI**
- Users should be able to:
  - Select dates directly from calendar
  - View team availability visually
- Avoid form-only UI; calendar is mandatory

---

## User Setup (for MVP)
Since authentication is not implemented:

- Use dummy users:
  - 5 Employees
  - 1 Manager

Example:
- User1, User2, User3, User4, User5 → Employees
- Manager1 → Manager

---

## Constraints
- No authentication or login system
- No external integrations
- No notifications for now

---

## Development Guidelines
- Follow PRD, user stories, and acceptance criteria strictly
- Do NOT assume extra features
- Build backend APIs first, then UI
- Keep code modular and simple

## Goal
Build a working MVP first, then iterate with improvements.
